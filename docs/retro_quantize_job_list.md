# レトロ減色仕様 詰めJOB LIST

このドキュメントは、会話しながら仕様を詰めるための未決定事項一覧である。

- 列は `ID / 項目 / 推奨 / 結論` とする
- `結論` 欄は、確定した時点で追記する
- `推奨` は初版実装を前に進めやすい案を優先する

| ID | 項目 | 推奨 | 結論 |
| --- | --- | --- | --- |
| JOB-001 | 透過画素の扱い | 完全透明画素は処理対象外とし、半透明画素は alpha を考慮した有効画素として扱う | |
| JOB-002 | 輪郭抽出で使う輝度の定義 | 初版は RGB から単純な輝度値を作り、その輝度画像に対して Sobel を適用する | |
| JOB-003 | 彩度の定義 | 初版は軽量な RGB ベースの簡易彩度指標を使う | |
| JOB-004 | 「局所的に暗い側に属する」の判定方法 | 勾配の両側比較で、より暗い側のみを輪郭候補として採用する | |
| JOB-005 | 細線優先の具体実装 | 初版は 1〜2px の連結成分を残しやすい後処理で代替する | |
| JOB-006 | 輪郭レイヤー後処理の初期セット | 小ノイズ除去、孤立点除去、細線連結を最小セットにする | |
| JOB-007 | 半解像度セルの目標色の算出方法 | セル内 2x2 の平均色を基本とし、輪郭被覆率が高いセルは誤差重みを下げる | |
| JOB-008 | パレット生成に使う画素の重み付け | 輪郭で強く覆われる領域は重みを下げ、色面を優先して学習する | |
| JOB-009 | 初版のパレット生成アルゴリズム | 初版は K-Means を第一候補にする | |
| JOB-010 | 2色混合の見え色計算 | 初版は linear RGB で混合色を作り、誤差評価は RGB または Lab に差し替え可能な構造にする | |
| JOB-011 | 候補生成の探索範囲 | 近傍単色上位 N 色から色ペア候補を作り、全探索は避ける | |
| JOB-012 | Confidence の正規化式 | 1 位候補と 2 位候補の誤差差を線形で 0..255 に写像する | |
| JOB-013 | ICM の近傍定義 | 初版は 4 近傍を採用する | |
| JOB-014 | 隣接整合 `S_ij` の初期項目 | 色相飛び、明度飛び、無意味な色ペア変化を先に入れ、位相衝突は後回しにする | |
| JOB-015 | 輪郭保護 `O_i` の具体則 | 輪郭近傍では `Mix50` を不利にし、単色または 75/25 系を優先する | |
| JOB-016 | レトロ優先 `R_i` の具体則 | ベタ塗り優先と過剰ディザ抑制を初期実装に入れる | |
| JOB-017 | パレット再学習の初版範囲 | 初版では簡易版または未実装でもよく、差し替えやすい分離を優先する | |
| JOB-018 | PSD 出力の実装方式 | 初版は後回しにし、先に内部データ構造を PSD 拡張しやすい形で固める | |
| JOB-019 | 公開 API の入口 | `IRetroQuantizer` 相当の単一入口を用意し、UI 側はそこだけを呼ぶ構成にする | |
| JOB-020 | 画像入力と出力の所有権 | 入力 `Bitmap` は呼び出し側所有、結果 `Bitmap` は結果オブジェクト所有として扱う | |
